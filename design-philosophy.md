# DESIGN PHILOSOPHY
# PROJECT: LA AVENTURA DE YAGUARÁ
# PURPOSE: Define the non-negotiable principles guiding all decisions
# AUDIENCE: AI system (Claude), designers, developers, educators

#####################################################################
# 0. WHY A DESIGN PHILOSOPHY IS REQUIRED
#####################################################################

This project is not a content library, an app, or a game with lessons.

It is:
- A learning ecology
- A mythic system
- A long-term relationship between learner and language

Therefore, decisions cannot be driven by:
- Efficiency alone
- Engagement metrics alone
- Content volume alone

They must be driven by a coherent worldview.

#####################################################################
# 1. CORE PHILOSOPHY (ONE SENTENCE)
#####################################################################

Language is not information.
Language is a living force that shapes relationships, memory, and reality.

Everything else follows from this.

#####################################################################
# 2. FUNDAMENTAL BELIEFS
#####################################################################

BELIEF 1: Language is Action
- Words do things
- Saying something changes the world
- Learning happens through meaningful use, not explanation

BELIEF 2: Learning is Cyclical
- Knowledge returns
- Mastery comes from revisiting, not finishing
- Progress is spiral, not linear

BELIEF 3: Meaning Precedes Form
- Learners feel meaning before analyzing structure
- Grammar emerges from use
- Rules are discovered, not delivered

BELIEF 4: The World is Alive
- Rivers, mountains, animals are agents
- Environment responds to language
- Learning is relational, not abstract

BELIEF 5: Responsibility Grows with Fluency
- Early language = naming
- Advanced language = ethical consequence
- Mastery implies care

#####################################################################
# 3. WHAT WE DESIGN (POSITIVE PRINCIPLES)
#####################################################################

WE DESIGN:
- Familiar play patterns with deep meaning
- Few mechanics, reused many times
- Silence, waiting, and listening as valid states
- Consequences instead of scores
- Calm interfaces that respect attention
- Systems that feel grown, not built

WE PRIORITIZE:
- Retention over speed
- Depth over breadth
- Agency over completion
- Return over novelty

#####################################################################
# 4. WHAT WE REFUSE TO DESIGN (NEGATIVE PRINCIPLES)
#####################################################################

WE REFUSE:
- Drills disguised as games
- Linear level ladders
- Constant rewards, points, streaks
- Timers and pressure
- Excessive text explanation
- "Wrong answer" punishment loops

WE DO NOT:
- Rush learners
- Treat language as trivia
- Separate language from culture
- Reduce indigenous knowledge to decoration

#####################################################################
# 5. DESIGN AXES (EVERY FEATURE MUST ALIGN)
#####################################################################

Every design decision must align on three axes:

AXIS 1 — COGNITIVE
- Does this support memory, transfer, and understanding?
- Is difficulty increasing through meaning, not speed?

AXIS 2 — MYTHIC
- Does this respect cyclical time?
- Does the world respond like a living system?

AXIS 3 — PEDAGOGICAL
- Is language used as action?
- Does this map to real communicative ability?

If a feature fails one axis, it is redesigned or removed.

#####################################################################
# 6. PROGRESSION PHILOSOPHY
#####################################################################

Progression is NOT:
- Completing modules
- Unlocking levels
- Accumulating points

Progression IS:
- Returning with deeper understanding
- Being trusted with more responsibility
- Seeing familiar things differently

The learner does not "finish" the world.
The learner becomes part of it.

#####################################################################
# 7. ASSESSMENT PHILOSOPHY
#####################################################################

We do not test language.

We observe:
- What the learner can do
- How they choose to speak
- Whether they can repair, negotiate, and adapt

Assessment is embedded in play.
Feedback is environmental, not judgmental.

#####################################################################
# 8. ROLE OF THE LEARNER
#####################################################################

At early stages:
- The learner is a listener and namer

At intermediate stages:
- The learner is a storyteller and negotiator

At advanced stages:
- The learner is a guardian and mediator

Fluency = expanded agency + expanded responsibility.

#####################################################################
# 9. ROLE OF THE SYSTEM
#####################################################################

The system is not a teacher.

The system is:
- A world
- A set of rules
- A memory
- A mirror

It presents situations.
The learner responds with language.
The world reacts.

#####################################################################
# 10. FINAL DESIGN STATEMENT
#####################################################################

This project treats language learning as:
- A cultural act
- A relational practice
- A lifelong spiral

We are not building a faster way to learn Spanish.
We are building a better way to belong to it.

END OF DESIGN PHILOSOPHY.
